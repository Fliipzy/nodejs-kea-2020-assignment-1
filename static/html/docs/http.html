HTTP

HyperText Transfer Protocol (HTTP) is the protocol used by the World Wide Web (WWW), to define how data is formatted and transmitted between clients and web-servers.

If you've ever used a browser before you'll without a doubt have used HTTP. Something as simple as navigating to google.com is actually sending a HTTP GET request to one of googles web servers and asking for all the stuff needed to render their website.

If you're using the Chrome browser you can click Ctrl + Shift + I to open the DevTools window. Inside this window you can inspect a websites content: html, css & javascript. You can also navigate to the Network tab, to see all the network activity, including HTTP requests.

GET method

As i mentioned above we use the GET request method wether we know it or not, when browsing the web:
When we load websites, look at google images etc. It's fair to say the GET method is the most common HTTP method by far.

When we use the GET method it means we as clients requests data from a resource (web-server).
GET requests should only retrieve data, not send it. I use the word should because how HTTP is implemented by people in their web applications can differ.

POST method

A POST method on the other hand should send data. Examples of sending POST requests is whenever you log into some site: facebook, youtube etc.
When you fill out the textfields that requires your username and password and hit the login button a POST request is/should be sent.
